# Reporting Roadmap

## Phase 1

- In-game live meter
- JSONL event output
- External HTML report ingestion
- Damage, defense, risk, support, efficiency, and awards from logged events

## Phase 2

- Session browser inside the report viewer
- Per-act, per-floor, per-combat drilldowns
- Damage source breakdown by card/relic/orb/poison/summon
- Support attribution views
- Compare players across multiple runs

## Phase 3

- Optional shared upload target
- Friends-only leaderboards
- Patch/ascension/party-size filters
- Replay-style combat timeline
